## 公文元素清单

| 序号 | 元素名称    | 对应方法                       | 是否必需              | 格式示例                                                                          |
| -- | ------- | -------------------------- | ----------------- | ----------------------------------------------------------------------------- |
| 1  | 集团名称    | `add_group()`              | 是。                | 固定是汇川技术                                                                       |
| 2  | 发文机关标志  | `add_issuer_mark()`        | 是                 | 单独一行，XXXX文件就是发文机关                                                             |
| 3  | 发文字号    | `add_document_header()`    | 是，如果有签发人，则和签发人是一行 | XXX〔2022〕X号，典型标志是六角括号                                                         |
| 4  | 签发人     | `add_document_header()`    | 否                 | 签发人：XX，或者签发人：X X                                                              |
| 5  | 版头分隔线   | `add_black_separator()`    | 是                 | 分割线不用管                                                                        |
| 6  | 密级      | `add_doc_classification()` | 是                 | 【密级：XXXX】                                                                     |
| 7  | 公文标题    | `add_title()`              | 是                 | 关于XXXX的通知，或者关于XXX的通报，或者关于XXX的公示                                               |
| 8  | 一级标题    | `add_heading_level1()`     | 否                 | 如果段落以"一、"、"二、"..."十一、"等中文数字开头，就是一级标题                                          |
| 9  | 二级标题    | `add_heading_level2()`     | 否                 | 如果段落以"（一）"、"（二）"..."（十一）"等中文数字括号开头，就是二级标题                                     |
| 10 | 三级标题    | `add_heading_level3()`     | 否                 | 如果段落以"1."、"2."..."100."等阿拉伯数字加点开头，就是三级标题                                      |
| 11 | 四级标题    | `add_heading_level4()`     | 否                 | 如果段落以"（1）"、"（2）"..."（100）"等中文数字括号开头，就是四级标题                                    |
| 12 | 五级标题    | `add_heading_level5()`     | 否                 | 如果段落以"①"、"②"..."⑳"等圈号数字开头，就是五级标题                                              |
| 13 | 正文段落    | `add_body_paragraph()`     | 是                 | 没有特殊格式                                                                        |
| 14 | 结尾语     | `add_closing()`            | 是                 | 特此通知。或者 特此公示。或者特此通报。                                                         |
| 15 | 附件说明    | `add_attachment_note()`    | 否                 | 有“附件：”就是附件区域，有可能一行，多个文件的时候有多行。多行的时候也以阿拉伯数字加点开头，例如：1. 《ATH事业群组建方案》，不要和三级标题搞混了。 |
| 16 | 成文日期    | `add_issue_date()`         | 是                 | 二〇二二年七月二十七日，读到这种就是成文日期                                                        |
| 17 | 主送机关    | `add_main_send()`          | 是                 | 主送：汇川全员，典型表示是主送：XXX                                                           |
| 18 | 抄送机关    | `add_copy_send()`          | 否                 | 抄送：朱兴明总裁。典型标识是  抄送：XXX                                                        |
| 19 | 印发机关/日期 | `add_print_info()`         | 是                 | 汇川技术总裁办    2022年7月27日印发，这两个在一行，是一个文件最后的内容了                                    |

***

除了标题和正文会交替出现，其他的元素都是从上到下的顺序出现的，不会乱了顺序。

1、附件说明结束的标志是什么？->读到成文日期，例如：二〇二五年三月十六日，就说明附件的内容已经完了
2、如何区分附件说明和三级标题？->两者都是"数字."开头，我的想法是： - 如果前面刚读到"附件："，那就是附件说明. - 否则就是三级标题，你的想法是对的。
3、正文中如果出现"特此通知"会误判吗？->不会，因为"特此通知/特此通报/特此公示"是结尾语，不是正文段落。并且只有这四个字加上一个句号，占一行，没有其他内容。
4、发文机关标志的判断？->如果段落以"XXXX文件"开头，就是发文机关标志。“文件”这两个字之后没有其他内容，也没有标点符号。实际上，文件的第二行内容就是发文机关，这是固定的，所以你读到的第二行内容就是发文机关。
5、主送机关"汇川全员"是固定值吗？->不是，不是固定的。抄送：朱兴明总裁。这个也不是固定的。
6、表格序号8有个小错误->是8，我改过来了
7、读到的分隔线都不用管
